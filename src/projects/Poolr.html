Poolr

The Problem:

Parents often struggle to manage daily school drop-offs and pick-ups due to tight schedules.

😰 Relying on public transport or paid drivers can be costly, unreliable, or unsafe.

👀 There’s no easy way to coordinate rides with other trusted parents in the same area.

The Solution – Poolr:

🚗 Poolr is a ridesharing app built exclusively for parents of school-going kids.

👨‍👩‍👧‍👦 It connects verified parents to organize safe, scheduled rides to and from school.

📍 With live tracking, secure tipping, and a trust-based network, Poolr makes school rides simpler, safer, and more efficient for everyone.

Key Features

  • 🛡️ Verified Parent Network

    Every user goes through a verification process, ensuring only trusted parents have access to the app—creating a safe, closed community.

  • 📅 Ride Scheduling & Coordination

    Parents can easily schedule pickups and drop-offs with other parents heading to the same school in advance, choose who’s driving, and manage their child’s commute with just a few taps.

  • 📍 Live Ride Tracking

    Real-time GPS tracking allows parents to monitor rides, ensuring peace of mind and full transparency throughout the commute.

  • 💸 Tip & Payment Integration

    Once the ride is completed, tipping is easy via Apple Pay, Stripe, or card—automating rewards for helping parents and reducing hassle.

Architecture

Technologies Used

1. Flutter

We use Flutter to build a beautiful, consistent, and high-performance mobile app for both Android and iOS. By using a single codebase, we ensure faster development cycles, lower maintenance overhead, and a consistent user experience across platforms.

2. AWS API Gateway

API Gateway acts as the entry point for all client-server interactions. It securely routes requests from the app to backend services, ensuring smooth communication between the front end and the serverless backend.

3. AWS Lambda

Our backend logic is powered by AWS Lambda, a serverless computing service. This eliminates the need for managing servers and allows Poolr to scale on demand based on user activity. This is majorly used for generating Payment URL for Stripe.

4. Stripe

Stripe is integrated to handle donations. It enables secure, flexible, and globally trusted payment processing—supporting both one-time and recurring donations via credit/debit cards.

5. Webhooks

We use webhooks for real-time event communication—such as receiving instant payment status updates from Stripe or notifying the system of a completed transaction. This keeps the app responsive and up-to-date.

6. Admin Portal

The Poolr admin portal is a web-based tool used by internal teams to monitor rides, manage emergencies, handler riders, validating documents, review analytics, provide transaction details, and maintain platform integrity. It provides complete backend visibility and control over the platform’s operations.

7. Firebase

Firebase is integrated into our application to manage user authentication securely and efficiently. It also powers real-time notifications, allowing users to receive instant updates and alerts, thereby enhancing the overall responsiveness and interactivity of the app.

8. Google Maps

Google Maps is utilized in our application to provide accurate geolocation services and interactive map functionalities. It enables features such as location tracking, address lookup, and route visualization, enhancing user experience through seamless navigation.

Architecture

Technologies Used

1. Flutter

We use Flutter to build a beautiful, consistent, and high-performance mobile app for both Android and iOS. By using a single codebase, we ensure faster development cycles, lower maintenance overhead, and a consistent user experience across platforms.

2. AWS S3 (Simple Storage Service)

All media assets, including NPO videos, images, and thumbnails, are stored on Amazon S3. This allows for secure, scalable, and cost-effective media storage with high availability, ensuring fast video delivery to users.

3. AWS API Gateway

API Gateway acts as the entry point for all client-server interactions. It securely routes requests from the app to backend services, ensuring smooth communication between the front end and the serverless backend.

4. AWS Lambda

Our backend logic is powered by AWS Lambda, a serverless computing service. This eliminates the need for managing servers and allows GTZP to scale on demand based on user activity‚Äîwhether it's a few users or thousands.

5. Stripe

Stripe is integrated to handle donations. It enables secure, flexible, and globally trusted payment processing‚Äîsupporting both one-time and recurring donations via credit/debit cards.

6. CloudFront (Content Delivery Network)

To ensure that video content and other media load quickly no matter where the user is located, we use Amazon CloudFront to deliver content through edge locations globally. This improves speed and reliability.

7. Webhooks

We use webhooks for real-time event communication‚Äîsuch as receiving instant payment status updates from Stripe or notifying the system of a completed transaction. This keeps the app responsive and up-to-date.

8. Admin Portal

The GTZP admin portal is a web-based tool used by internal teams and NPOs to manage content, review analytics, approve donations, and maintain platform integrity. It provides complete backend visibility and control over the platform‚Äôs operations.

Supabase Guide

Build powerful apps with the open-source Firebase alternative

#PostgreSQL #Realtime #OpenSource

Why Choose Supabase?

Supabase combines the power of PostgreSQL with a suite of tools that make backend development accessible while maintaining enterprise-grade capabilities.

PostgreSQL Database

Full-featured relational database with row-level security, extensions, and a powerful SQL editor built-in.

Authentication

Secure user management with email/password, social logins, and JWT tokens integrated with your database.

Realtime API

Listen to database changes in real-time via websockets, perfect for chat apps and live data.

Supabase vs Firebase

Feature Supabase Firebase
Database PostgreSQL (relational) Firestore (NoSQL)
Pricing Pay-as-you-go, generous free tier Pay-as-you-go, free tier with limits
Open Source Yes (MIT License) No
Hosting Self-host or use Supabase cloud Google Cloud only
Query Language SQL (with client libraries) Custom query language

Quick Start Guide

1. Install the client library


// Using npm
npm install @supabase/supabase-js

// Using yarn
yarn add @supabase/supabase-js

2. Initialize the client


import { createClient } from '@supabase/supabase-js'

const supabaseUrl = 'YOUR_SUPABASE_URL'
const supabaseKey = 'YOUR_SUPABASE_KEY'
const supabase = createClient(supabaseUrl, supabaseKey)

3. Make your first query


async function getUsers() {
  const { data, error } = await supabase
    .from('users')
    .select('*')
  
  if (error) console.error(error)
  else console.log(data)
}

Authentication Example

Sign up a new user


async function signUp(email, password) {
  const { user, error } = await supabase.auth.signUp({
    email: email,
    password: password,
  })
  
  if (error) throw error
  return user
}

Sign in a user


async function signIn(email, password) {
  const { user, error } = await supabase.auth.signIn({
    email: email,
    password: password,
  })
  
  if (error) throw error
  return user
}

Protected route example


async function getProtectedData() {
  const { data: { user }, error } = await supabase.auth.getUser()
  
  if (error || !user) {
    // Redirect to login
    return
  }
  
  // User is authenticated
  const { data, error: dbError } = await supabase
    .from('protected_data')
    .select('*')
    .eq('user_id', user.id)
  
  if (dbError) throw dbError
  return data
}

Advanced Features

Row Level Security

Supabase leverages PostgreSQL's Row Level Security to control exactly what data users can access. 


-- Enable RLS on a table
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;

-- Create a policy
CREATE POLICY "Users can view their own profile" 
ON profiles FOR SELECT 
USING (auth.uid() = user_id);

Realtime Subscriptions

Listen to database changes in real-time with Supabase's websocket connection. 


const subscription = supabase
  .channel('public:messages')
  .on('postgres_changes', {
    event: 'INSERT',
    schema: 'public',
    table: 'messages'
  }, (payload) => {
    console.log('New message:', payload.new)
  })
  .subscribe()
